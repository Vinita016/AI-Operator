What is the single operation an LLM performs?
It predicts the most likely next token, over and over. Everything else — the brilliance and the failures — follows from that.
Why do LLMs hallucinate?
They optimize for plausible continuations, not truth. Sounding confident and being correct are different objectives.
What does temperature control?
Randomness in token selection. Low = consistent and safe. High = creative, eventually incoherent.

Prompting — Command the Machine

GOAL → Never write a lazy prompt again. The gap between a casual user and an operator is 90% prompt discipline. 0/5 drills

Here's the uncomfortable truth: your prompts are probably trash. "Write me an email" gets you generic mush because you gave the model nothing to work with. Great prompts have five components: role, context, task, format, constraints. The resources below are the actual playbooks used by the labs that build these models — not influencer threads.

Interactive · Prompt Grader

Grade your prompt like an operator would

Paste a real prompt you'd actually use. The grader checks it against the five components of a professional prompt. Under 60? Rewrite it. This is the drill that changes your daily output the fastest.

  • Assigns a role or persona ("You are a...", "Act as...")
  • Provides context or background (audience, situation, "for", "about")
  • Specifies output format (bullets, table, length, tone, structure)
  • Includes an example or reference to imitate
  • Sets constraints (what to avoid, limits, "don't", "must")
  • Long enough to carry real detail (40+ words)

Name the 5 components of an operator prompt.
Role, context, task, format, constraints. Add an example of great output and you're in the top tier.
What is few-shot prompting?
Showing the model 1–3 examples of the output you want so it imitates the pattern instead of guessing.
Output came back generic. First move?
Add specificity: role + context + format + one example. Never just hit retry and hope.

BRIEF — Custom AI Assistant
  1. Pick one recurring task (weekly report, client emails, meeting prep).
  2. Write a system prompt using your Phase 02 skills: role, context, format, constraints, 2 examples of great output.
  3. Build it as a Claude Project or custom GPT. Feed it your reference docs.
  4. Use it daily for a week. Log what fails. Fix the prompt each time.
  5. Ship test: give it to one colleague. If they keep using it, you shipped.
BRIEF — AI Workflow Automation
  1. Map one multi-step task: e.g., "new form response → AI summarizes & drafts reply → posts to Slack/email."
  2. Sign up for Zapier or Make (free tiers work). Connect your trigger app.
  3. Add an AI step with a tight, engineered prompt. Test with 5 real inputs.
  4. Add the output step. Run it live for a week; track errors and time saved.
  5. Ship test: it runs 10 times without you touching it.
BRIEF — AI Web App
  1. Pick a one-feature idea: résumé critiquer, contract explainer, workout generator. One feature. Not a platform.
  2. Get an API key (Anthropic or OpenAI). Read the quickstart — it's shorter than you think.
  3. Use AI itself to scaffold the app (ask it for a simple Python/JS app that calls the API). Debug by pasting errors back in.
  4. Deploy free on Vercel, Replit, or Streamlit Cloud.
  5. Ship test: 5 strangers use it and one gives you feedback you didn't expect.
